# Contributing to This Repository

Thank you for your interest in contributing to this repository. Please note that
this repository contains generated code. As such, we do not accept direct
changes or pull requests. Instead, we encourage you to follow the guidelines
below to report issues and suggest improvements.

## How to Report Issues

We use GitHub issues to track bugs and enhancements. Found a bug in the source
code? Want to propose a new feature or enhancement?

Before you submit your issue, search the [issues repository](https://github.com/gravitee-io/issues/issues);
maybe your question has already been answered.

[Report a new issue](https://github.com/gravitee-io/issues/issues/new/choose)

> Issues are only for bug reports, enhancement requests, and new feature
> requests. For general questions and discussions, use the
> [Community forum](https://community.gravitee.io?utm_source=contributing).

Providing the following information will help us deal with your issue quickly:

* **Tag the issue with `project: AM`**
* **Overview of the issue**: describe the issue and why this is a bug for you.
* **Gravitee AM version(s)**: possible regression? (See [COMPATIBILITY.md](COMPATIBILITY.md) for the supported provider-to-AM mapping.)
* **Terraform CLI version**: e.g. `1.9.*`
* **Terraform Provider version**: e.g. `0.0.1`
* **Do you have logs, traces, server error stack traces?** Add these to the issue's description.
