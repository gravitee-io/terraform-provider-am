# CLAUDE.md

Guidance for Claude Code working in this repository.

## What This Is

Terraform provider for **Gravitee Access Management (AM)**, generated by
[Speakeasy](https://www.speakeasy.com/) from `automation-api-oas.yaml`
plus the overlays in `.speakeasy/overlays/`. The vast majority of the Go is
generated — **never edit files carrying `// Code generated by Speakeasy`**.
All behavioural change happens through overlays + regen.

## Build & Test Commands

```bash
# Regenerate the provider, SDK, docs, examples
make speakeasy

# Apply overlays and lint the merged OAS (zero errors, zero warnings goal)
make lint
make lint-fix          # fix Terraform fmt

# Just rebuild the merged-OAS artifact (.speakeasy/output/computed.json) without linting
make merge-overlays

# Unit tests (no AM instance required)
make unit-tests

# Build the provider binary
go build

# Acceptance tests — not yet wired up
make acceptance-tests   # currently exits 1 with a TODO
```

The Speakeasy CLI must be authenticated (`speakeasy auth login`). The pinned
version lives in `.speakeasy/workflow.yaml` (`speakeasyVersion`).

## Architecture

### Code generation flow

1. OAS lives at `automation-api-oas.yaml` (currently a hand-copied
   snapshot; no upstream sync yet)
2. Overlays under `.speakeasy/overlays/` patch the OAS — applied in the order
   declared in `.speakeasy/workflow.yaml`
3. Speakeasy emits the SDK (`internal/sdk/`), provider (`internal/provider/`),
   docs (`docs/`), and examples (`examples/`)
4. Pipeline configured by `.speakeasy/gen.yaml` (target settings,
   environment-variable mappings, naming) and `.speakeasy/workflow.yaml`
   (sources + overlay chain)
5. `.speakeasy/gen.lock` is the source of truth for the next release version
   (read by the release workflow)

### Provider structure (`internal/provider/`)

- **4 managed resources**: `am_domain`, `am_certificate`,
  `am_identity_provider`, `am_reporter` — certificates/IdPs/reporters are
  nested under a Domain
- **4 data sources** mirror them
- Each resource: `*_resource.go` (CRUD) + `*_resource_sdk.go` (Terraform ↔
  SDK type mapping)
- `provider.go` — provider schema, auth wiring, defaulting
- `provider_test.go` — sanity-only unit tests for schema + registration

### Authentication

**Bearer-only.** AM's Automation API accepts a service-account access token
or JWT. There is no Cloud token flow and no basic-auth fallback (unlike
APIM). The token comes from `AM_SA_TOKEN` or the `bearer_auth` provider
attribute.

`organization_id` / `environment_id` default to `"DEFAULT"` (set via the
OAS `schema.default` in `common/param.yaml`, which Speakeasy turns into
the defaulting block in the generated `provider.go`). Override via
`AM_ORG_ID` / `AM_ENV_ID` env vars or provider config for multi-tenant
installations.

### Overlays (`.speakeasy/overlays/`)

| File | Purpose |
|---|---|
| `common/oas.yaml` | Root-level: `x-speakeasy-retries`, `x-speakeasy-globals` promoting org/env, README/title rewrites |
| `common/param.yaml` | Synthesises `components.parameters.{orgIdParam, envIdParam}` (the raw OAS has none), names them `organizationId`/`environmentId`, replaces every inline `orgId`/`envId` with a `$ref` |
| `common/schema.yaml` | `x-speakeasy-param-computed: false` globally (cleaner plans), ignore the `Error` schema |
| `domain.yaml`, `certificate.yaml`, `identities.yaml`, `reporter.yaml` | Per-entity: `x-speakeasy-entity`, three `x-speakeasy-entity-operation` mappings (create-or-update/read/delete), `key`-path-param rename, sensitive marking for `configuration` |

### AM-specific gotchas (worth knowing before editing overlays)

1. **Body `key` vs path `{xxxKey}` mismatch.** The AM OAS uses `key` in
   request bodies but `{domainKey}`/`{certKey}`/`{identityKey}`/`{reporterKey}`
   in URL paths. Speakeasy can't auto-correlate, so each per-resource
   overlay renames the path parameter to `key` via
   `x-speakeasy-name-override: key` on the GET and DELETE operations. This
   is the fix for the *"required parameter not defined in create"* lint
   error you'll hit if you remove these annotations.

2. **`null = remove` doesn't work in Speakeasy overlays.** Per the
   OpenAPI Overlay 1.0 spec (§3.4), `update` with a `null` value should
   delete the field. Speakeasy's implementation merges the literal `null`
   (or the string `"null"`) instead. The workaround in `common/param.yaml`:
   add the `$ref` via `update`, then issue individual `remove:` actions
   for each sibling field. `name` is removed last so the JSONPath filter
   `[?@.name=='orgId']` keeps matching through the chain. Don't try to
   shortcut this with a single `update` containing `null`s.

3. **The merged OAS lives at `.speakeasy/output/computed.json`** (gitignored).
   When an overlay isn't producing what you expect, run `make merge-overlays`
   and read that file — it's exactly what Speakeasy's codegen sees.

## Changing provider schema behaviour

Edit or add overlays, never the generated files. Overlay annotations to
reach for first:

- `x-speakeasy-param-computed: true` — mark a field as computed
- `x-speakeasy-param-suppress-computed-diff: true` — add `SuppressDiff`
  plan modifier (only works on Unknown plan values)
- `x-speakeasy-param-sensitive: true` — mark Terraform attribute Sensitive
- `x-speakeasy-name-override: <name>` — rename a parameter or schema
- `x-speakeasy-terraform-ignore: true` — drop from generated Terraform state
- `x-speakeasy-entity` / `x-speakeasy-entity-operation` — map an OAS schema
  + operations onto a Terraform resource lifecycle

There are no `internal/planmodifiers/`, `internal/validators/`, or
`internal/customtypes/` packages in this repo yet — add them when an
overlay actually needs to reference custom Go.

## Conventions

- **CONTRIBUTING.md is hand-maintained** (mirroring APIM's pattern) and
  listed in `.genignore` so Speakeasy won't overwrite it. Don't replace
  it with the Speakeasy default.
- **`USAGE.md` and `examples/README.md` are auto-generated and auto-deleted**
  by `make speakeasy`. This suppresses the README's "Testing the provider
  locally" section.
- **Import IDs are JSON-encoded composite keys** (e.g.
  `{"domain_key":"...","env_id":"...","key":"...","org_id":"..."}`). See
  any generated `examples/resources/am_*/import.sh`.
- **Release versioning is automatic** — Speakeasy bumps
  `.speakeasy/gen.lock`'s `management.releaseVersion`. The release workflow
  tags whatever's in that field. Provider version is decoupled from AM
  product versions.
- **No public registry push yet.** The `registry:` block in
  `workflow.yaml` is intentionally absent until the Speakeasy workspace
  source-name is confirmed (see plan notes / DEVELOPMENT.md).
